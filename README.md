# Guardian of Truth: Hallucination Detection via Hidden State Probing

Competitive ML solution for detecting factual hallucinations in GigaChat answers. The detector uses internal model signals from a single forward pass: hidden-state probes, contrast directions, PCA-compressed representations, and logit-based uncertainty features.

**Hackathon result:** 3rd place in the Sber hallucination detection track.  
**Final/private PR-AUC:** ~0.8541.  
**Public validation PR-AUC:** 0.8493 with the config in `configs/best_config.json`.

## Why This Project Is Interesting

The task was to predict whether an answer generated by `ai-sage/GigaChat3-10B-A1.8B-bf16` contains a factual hallucination. The detector had to stay fast: the competition allowed using the original GigaChat forward pass for internal signals, but disallowed runtime external APIs, RAG, and LLM-as-a-judge style verification.

The final approach treats hallucination detection as a representation probing problem:

- capture hidden states from early and middle transformer layers during teacher forcing;
- compute uncertainty features from answer-token logits;
- learn contrast directions between correct and hallucinated answers;
- compress probe vectors with PCA;
- train a lightweight logistic regression classifier for fast inference.

## Method

1. **Teacher forcing**
   Feed `prompt + model_answer` through GigaChat and read activations over the answer span.

2. **Hidden-state probes**
   Use layers `L3`, `L9`, and `L15`. Early layers were especially useful, which matched the working hypothesis that factual information appears before later reasoning/generation layers distort it.

3. **Contrast directions**
   For each probe representation, compute the normalized difference between mean hallucinated and mean correct activations. The projection onto this direction becomes a compact feature.

4. **Uncertainty scalars**
   Add 18 logit-derived features: token log-probability statistics, entropy, top-k probabilities, margins, answer length, perplexity proxy, and low-confidence token share.

5. **Synthetic training data**
   Generate additional factual questions, answer them with GigaChat, and verify labels offline. These labels are used only for training data construction, not as part of the runtime detector.

6. **Lightweight classifier**
   Combine scalars, PCA-compressed probes, and contrast projections, then train `LogisticRegression`.

## Results

| Experiment | Public PR-AUC |
| --- | ---: |
| Uncertainty-only baseline | ~0.80 |

## Quick Start

Python 3.10+ and a CUDA GPU are recommended. The hackathon environment used an RTX 5090 32 GB; final evaluation targeted an A100.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
export PYTHONPATH="$PWD/src"
```

If the cached features are available, train/evaluate the lightweight classifier without recomputing all GigaChat activations:

```bash
bash scripts/train.sh
```

To score a private benchmark file:

```bash
bash scripts/score_private.sh
```

Full feature extraction requires loading `ai-sage/GigaChat3-10B-A1.8B-bf16` and is GPU-heavy. The current public-facing tree keeps the stable scoring/classifier path and documents the generated artifacts; see [`features/README.md`](features/README.md) before regenerating feature caches.

## Public GitHub Checklist

Before pushing this repository publicly, remove competition-private files, local shell files, and large binary caches from the git index. The `.gitignore` already prevents new copies from being added, but files committed in the original hackathon snapshot remain tracked until removed.

```bash
git rm --cached .bash_history .bash_logout .bashrc .profile
git rm --cached data/bench/*private*.csv data/bench/*scores*.csv
git rm --cached data/generated/*.csv
git rm --cached features/*.npz
```

Keep the files locally if you need them for reproduction; the commands above only untrack them. If the data license permits publishing some benchmark/training files, document that explicitly in `data/README.md` before keeping them in the public repo.

## Notes

- Runtime detection does not call external APIs.
- No RAG or external knowledge base is used at inference time.
- No fine-tuning of GigaChat is required.
- The classifier is intentionally small, so the overhead after the GigaChat forward pass stays low.
